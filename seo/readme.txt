Описания у слоя
link: ссылка которая определяет содержание слоя согласно текущего адреса в адресной строке, без учёта сессий и тп

у seo link формируется согласно schema
у слоя link формируется согласно state

чтобы определить с какими параметрами показывается слой используется layer.state
и далее по полученному link ищется такойже link только сформированный уже согласно schema и если найден используются его keywords и тп

linktpl: если содержимое слоя зависит от адреса, то шаблон адреса


link:'?{crumb.parent.parent.name}/{crumb.parent.name}/{crumb.parent.name}',

Описание для главной страницы
name для админки определится автоматически 
link по умолчанию определяется по childs link:"?" это можно определить без активного адреса.. ###child### проверка что нет такого уровня
seo:{
	title:"Заголовок браузера очень важен",
	keywords:"ключевое слово, ещё одно слово",
	description:"Описание страницы"
}

Система приводит к виду
seo:{
	name:"Главная страница",
	link:"?",
	items:[{
		title:"Заголовок браузера очень важен",
		keywords:"ключевое слово, ещё одно слово",
		description:"Описание страницы"
	}]
}

Описание для раздела
seo:{
	title:"Блог",
	keywords:"Блог",
	description:"Блог"
}

Описание для группы страниц
seo:{
	schema:{type:"string"},
	link:"?Блог/{.}",
	defitems:"*blog.php?seo"
}

Описание для группы страниц c дефолтными значениями
seo:{
	schema:{type:"string"},
	link:"?Блог/{.}",
	defitems:"*blog.php?seo"
	items:[{
		title:"Блог",
		keywords:"Блог",
		data:"Статья",
		description:"Блог"
	}]
}

если есть schema обязательно должно быть либо items либо defitems
